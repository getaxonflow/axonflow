## How to Contribute

We welcome contributions from the community! Here's how to get started:

1. **Fork the repository** - Create your own copy
2. **Create a feature branch** - `git checkout -b feature/your-feature`
3. **Make your changes** - Follow our coding standards
4. **Write tests** - Ensure your changes are covered
5. **Submit a pull request** - Describe your changes clearly

See [CONTRIBUTING.md](CONTRIBUTING.md) for detailed guidelines.

## Types of Contributions

We value all types of contributions:

- **Code** - Bug fixes, features, performance improvements
- **Documentation** - Guides, examples, API docs, typo fixes
- **Testing** - Test cases, bug reports, quality assurance
- **Design** - UI/UX improvements, architecture proposals
- **Community** - Answering questions, helping other users
